Signal Desk

Start from a question or event, then use page context and focused evidence to make it easier to understand what you are reading. The current prototype supports business explanation paths and a narrow research-evidence flow.

Run analysis

The tool turns a concrete question or event into a small number of possible explanations, or into a short list of relevant research papers when that is the better fit.

Optional structured evidence

Add one business-style table if the page includes peer comparisons, quarterly metrics, or expectation-vs-actual data. The research-evidence path ignores this table field in the first slice.

Searches a small supported set of public catalogs: data.gov, NYC Open Data, and the Chicago Data Portal.

Guardrails
  • Start from a real question or event whenever possible
  • Structured tables are supporting evidence, not the default driver
  • Outputs are explanation paths or research evidence summaries, not conclusions
